# Patterns

Compositional design patterns for the HaulEx Driver App. Each pattern explains when to use it, what tokens to apply, and what to avoid.

## Index

- [[Sheet Patterns]] — Half sheet, Full sheet, Stack Button Sheet — when to use each
- [[Navigation Patterns]] — Navigation stack vs Tab Bar vs Sheet, nav bar rules
- [[Form Patterns]] — Input fields, states, validation, layout rules
- [[Empty State Patterns]] — Default empty, no results, loading, error states

---

## How to use these patterns

Before designing a new screen:

1. Identify which patterns apply (most screens involve navigation + at least one list or form)
2. Read the token spec in the relevant pattern note
3. Follow the Do / Don't section — these capture the most common mistakes

If a screen requires a new pattern not documented here, design it and add a new note.
